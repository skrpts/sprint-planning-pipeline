## Overview

The Scrum Guide is the definitive reference for Scrum framework practices. It defines the rules, roles, events, and artefacts that make up the Scrum methodology.

## Key Concepts

### Sprint Planning

Sprint planning initiates the sprint by laying out the work to be performed. The resulting plan is created by the collaborative work of the entire Scrum team.

### Sprint Goal

The sprint goal is the single objective for the sprint. It provides flexibility in terms of the exact work needed to achieve it.

### Definition of Done

A formal description of the state of the increment when it meets the quality measures required for the product. When a product backlog item meets the definition of done, an increment is born.

## Usage

Reference this source when breaking down epics, estimating stories, or facilitating sprint ceremonies to ensure alignment with Scrum best practices.
